## 1. Project Overview

Elder Watch detects:
- **Falls** (sudden horizontal body orientation)
- **Near-falls** (dangerous posture / high velocity)
- **Prolonged inactivity** (lying still > threshold seconds)

**Pipeline:**
```
Camera → Preprocess (CLAHE) → YOLO (person/fall bbox) →
MediaPipe Pose (33 landmarks) → Optical Flow (motion) →
Rule-based Classifier → Alert (SMS / Email / Telegram)
```

**Hardware:** Raspberry Pi 5 (4 GB) + Camera Module 3 (ceiling-mounted)
**Target:** 15–25 FPS, <30 ms/frame, <5W power, ~6 MB INT8 model

## 3. System Architecture

```
┌─────────────────────────────────────────────────────┐
│              Hardware Layer (RPi 5)                  │
│  Camera Module 3 (120°, ceiling-mounted @ 2-3m)     │
└───────────────────┬─────────────────────────────────┘
                    │ BGR frame
┌───────────────────▼─────────────────────────────────┐
│              Preprocessing                           │
│  • Resize to 640×640                                 │
│  • CLAHE (adaptive brightness equalization)          │
│  • Gaussian denoising                                │
│  • Optical Flow (Farneback, half-res)                │
└───────────────────┬─────────────────────────────────┘
                    │ processed frame
┌───────────────────▼─────────────────────────────────┐
│              YOLO Detection (INT8 TFLite)            │
│  • Input: 640×640 INT8                               │
│  • Classes: [0=person, 1=fall]                       │
│  • Output: bounding boxes + confidence               │
│  • Threshold: person≥0.50 / fall≥0.70               │
└───────────────────┬─────────────────────────────────┘
                    │ person bboxes
┌───────────────────▼─────────────────────────────────┐
│              MediaPipe Pose (model_complexity=0)     │
│  • Input: cropped person region                      │
│  • Output: 33 landmarks (x,y,visibility)             │
│  • Features: aspect_ratio, com_y, torso_angle,       │
│              knee_angle, velocity, head_below_hips   │
└───────────────────┬─────────────────────────────────┘
                    │ pose features
┌───────────────────▼─────────────────────────────────┐
│              Fall Classifier (State Machine)         │
│  score = Σ(rule hits): ≥3 → FALL                    │
│  Rules: aspect<2.0, com_y<0.45, torso>30°,          │
│         head_below_hips, velocity>0.08, YOLO_fall,  │
│         knee_angle<120°                              │
│  Temporal: confirmed over 3 consecutive frames       │
│  States: NORMAL → NEAR_FALL → FALL_DETECTED          │
│                              → INACTIVE_ALERT        │
└───────────────────┬─────────────────────────────────┘
              FALL_DETECTED / INACTIVE_ALERT
┌───────────────────▼─────────────────────────────────┐
│              Alert Manager (30s cooldown)            │
│  • Snapshot (320×240 JPEG)                           │
│  • Twilio SMS  / SMTP Email  / Telegram Bot          │
│  (background thread — non-blocking)                  │
└─────────────────────────────────────────────────────┘
```

## 4. Step-by-Step Implementation Guide

### Phase 0: Desktop Environment Setup

> Run all training on a machine with a CUDA GPU. Fine-tuning on CPU is possible but slow.

```bash
# Clone / navigate to project
cd elder_watch/

# Create virtual environment
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install training dependencies
pip install -r requirements.txt

# Verify GPU
python -c "import torch; print(torch.cuda.is_available())"
```

### Phase 1: Dataset Preparation

All datasets below are **free and open-source**. You do not need to pay for any of them.

#### Overview of available datasets

| # | Dataset | Images / Frames | Format | Download | Notes |
|---|---------|----------------|--------|----------|-------|
| 1 | **Roboflow Fall Detection** | 4,497 annotated | YOLO (ready) | Roboflow API / web ZIP | **Easiest — no video processing needed** |
| 2 | **URFD** (Univ. Rzeszow) | ~2,600 PNG frames | Raw frames | Direct ZIP, no login | 30 fall + 40 ADL sequences |
| 3 | **Le2i FDD** | ~8,000 frames from 221 videos | Video + annotation files | Kaggle (free account) | Home + coffee room scenes |
| 4 | **MCFD** (Univ. Montreal) | ~4,000 frames | Raw PNG frames | Direct download | 8 fall scenarios, 3 cameras |

Use **at least one**. Combining all four gives the best generalisation.

---

#### Option A — Roboflow Fall Detection (Recommended for fastest start)

> Pre-annotated, already split 70/20/10, YOLO format. No video extraction needed.

**Method 1 — Auto-download via Python (needs free API key):**
```bash
pip install roboflow
# Get your free API key at: https://app.roboflow.com (sign up → Settings → API)
python training/scripts/prepare_datasets.py \
    --download-roboflow \
    --api-key YOUR_ROBOFLOW_API_KEY \
    --out datasets/processed
```

**Method 2 — Manual ZIP download (no login required for public datasets):**
```
1. Go to: https://universe.roboflow.com/roboflow-universe-projects/fall-detection-ca3o8/dataset/4
2. Click "Download Dataset" → Format: YOLOv8 → Download ZIP
3. Extract to: datasets/raw/roboflow_fall/
```
```bash
python training/scripts/prepare_datasets.py \
    --roboflow datasets/raw/roboflow_fall \
    --out datasets/processed
```

#### Option D — Combine all sources (Best accuracy)

```bash
python training/scripts/prepare_datasets.py \
    --roboflow  datasets/raw/roboflow_fall \
    --urfd      datasets/raw/urfd \
    --le2i      datasets/raw/le2i \
    --out       datasets/processed \
    --split     0.70 0.15 0.15
```

The script automatically:
- Extracts frames at 5 FPS from videos
- Applies per-frame FALL/PERSON labels from annotation files (Le2i)
- Augments fall frames (hflip + brightness jitter) if class-imbalanced
- Writes `training/configs/dataset.yaml`


### Phase 2: Fine-tuning YOLOv8

```bash
# Standard training (GPU)
python training/scripts/train_yolov8.py \
    --config training/configs/yolov8_fall.yaml \
    --data   training/configs/dataset.yaml \
    --device 0 \
    --epochs 50

# Resume from checkpoint
python training/scripts/train_yolov8.py \
    --resume runs/detect/elder_watch/weights/last.pt

# Monitor training
# TensorBoard: tensorboard --logdir runs/detect
```

**Output:** `models/finetuned/yolov8n_fall.pt`

**Expected metrics (YOLOv8n on these datasets):**
- mAP50 > 0.90
- Precision > 0.92
- Recall > 0.88

### Phase 3: Quantization (PTQ INT8)

> This step converts the ~25 MB FP32 model to ~6 MB INT8 TFLite for the RPi.

```bash
# Full PTQ pipeline
python training/scripts/quantize.py \
    --model models/finetuned/yolov8n_fall.pt \
    --data  training/configs/dataset.yaml \
    --out   models/quantized/yolov8n_fall_int8.tflite \
    --cal-size 1000

# Quick alternative (Ultralytics built-in):
# python -c "
# from ultralytics import YOLO
# m = YOLO('models/finetuned/yolov8n_fall.pt')
# m.export(format='tflite', int8=True, data='training/configs/dataset.yaml')
# "
```

**Expected quantization results:**

| Metric | FP32 | INT8 |
|--------|------|------|
| Size | ~6.2 MB | ~1.6 MB |
| mAP50 drop | — | <2% |
| Speed (RPi 5) | ~10 FPS | **20–25 FPS** |

#### Evaluate both models

```bash
python training/scripts/evaluate.py \
    --pt     models/finetuned/yolov8n_fall.pt \
    --tflite models/quantized/yolov8n_fall_int8.tflite \
    --data   training/configs/dataset.yaml
```

### Phase 4: Raspberry Pi Setup

#### 4a. Flash Raspberry Pi OS

1. Download **Raspberry Pi OS 64-bit Bookworm** from https://www.raspberrypi.com/software/
2. Flash to SD card using Raspberry Pi Imager
3. Enable SSH in imager settings
4. Boot and SSH in: `ssh pi@raspberrypi.local`

#### 4b. Transfer project files

```bash
# From your desktop machine:
rsync -avz --exclude='datasets/' --exclude='runs/' --exclude='venv/' \
    elder_watch/ pi@raspberrypi.local:~/elder_watch/

# Transfer the quantized model
scp models/quantized/yolov8n_fall_int8.tflite \
    pi@raspberrypi.local:~/elder_watch/models/quantized/
```

#### 4c. Run setup script on RPi

```bash
# SSH into RPi
ssh pi@raspberrypi.local

cd ~/elder_watch
chmod +x deployment/rpi_setup/setup_rpi.sh
./deployment/rpi_setup/setup_rpi.sh

# Reboot to apply changes
sudo reboot
```

#### 4d. Mount camera

The Camera Module 3 should be ceiling-mounted at **2–3 metres above the monitored area**
(staircase, living room) for the best overhead view.

Connect the camera via the CSI ribbon cable and verify:
```bash
libcamera-hello --list-cameras
v4l2-ctl --list-devices
```

```bash

sudo systemctl status elder-watch

```

## 5. Configuration Reference

Key settings in `configs/config.yaml`:

| Setting | Default | Description |
|---------|---------|-------------|
| `camera.fps` | 30 | Camera capture FPS |
| `camera.flip_vertical` | false | Set true for inverted ceiling mount |
| `yolo.confidence_threshold` | 0.50 | Person detection threshold |
| `yolo.fall_confidence_threshold` | 0.70 | Fall class threshold |
| `mediapipe.model_complexity` | 0 | 0=Lite(RPi), 1=Full, 2=Heavy |
| `classification.confirmation_frames` | 3 | Frames required to confirm fall |
| `classification.inactivity_threshold_seconds` | 5 | Seconds before INACTIVE alert |
| `alert.alert_cooldown_seconds` | 30 | Min seconds between alerts |

## 7. Troubleshooting

| Problem | Solution |
|---------|----------|
| `Cannot open camera` | Run `v4l2-ctl --list-devices`; check `/dev/video0` exists |
| MediaPipe slow on RPi | Ensure `model_complexity: 0` in config |
| TFLite model not found | Run quantize.py first; check `models/quantized/` path |
| High false positive rate | Increase `confirmation_frames` to 4–5 |
| No alerts received | Check Telegram bot token; verify cooldown setting |
| OOM crash on RPi | Reduce `camera.width/height` to 480×360 |
| systemd service fails | Check `journalctl -u elder-watch -n 50` for errors |
